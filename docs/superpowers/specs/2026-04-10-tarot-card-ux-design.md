# 타로카드 UX 설계

**날짜:** 2026-04-10
**상태:** 승인됨

---

## 개요

TOP3 도시 추천 결과를 타로카드 메타포로 재설계한다.
5장의 카드 뒷면을 보여주고, 유저가 직감으로 3장을 골라 뒤집는 경험을 제공한다.
나머지 2장은 잠금(유료 BM 대비). 공개된 3장 중 1장을 선택하면 타로 리더가 도시를 해설하는 상세 가이드(리딩)를 제공한다.

## 서비스 포지셔닝과의 연결

"어떤 노마드가 될지 모르는 거야. 우리가 찾아줄게."
→ 카드를 고르는 행위 자체가 자기 발견 경험. 타로의 "직감으로 끌리는 카드"가 NNAI의 "나 서사 강화" 필터와 일치한다.

---

## 전체 플로우

```
폼 완료
  → POST /api/recommend (top_n=5)
  → 로딩: "당신의 카드를 준비하고 있어요..."
  → [Stage 1] 덱 펼치기: 5장 뒷면 부채꼴 배치
  → 안내: "끌리는 카드 3장을 골라보세요"
  → 유저가 3장 탭 (선택 글로우 표시)
  → "카드 열기" CTA 버튼
  → [Stage 2] 3장 순차 플립 + 2장 잠금
  → 안내: "끌리는 도시를 하나 골라보세요"
  → 유저가 1장 탭
  → "리딩 받기" CTA (OAuth 게이트)
  → [Stage 3] 타로 리딩 페이지 (LLM 상세 가이드)
  → [Stage 4] 3장 비교 뷰
```

---

## Stage 1: 덱 펼치기

### 카드 배치
- **부채꼴 (Fan Spread)**: 5장이 테이블 위에 부채꼴로 펼쳐진 형태
- 카드 비율: 세로 2:3
- 중앙 카드가 수직, 좌우로 갈수록 기울어짐 (각 ±10°, ±20°)

### 셔플 애니메이션
1. 화면 중앙에 카드 5장이 겹쳐서 쌓여 있음
2. 0.3초 간격으로 한 장씩 부채꼴 위치로 이동 (Framer Motion stagger)
3. 전체 약 2초

### 카드 뒷면
- 완전 블라인드: 5장 모두 동일한 뒷면 디자인
- 힌트 없음 (대륙, 기후 등 어떤 정보도 표시하지 않음)
- 디자인: 딥 네이비 배경 + 골드 테두리 + 중앙에 NNAI 심볼 또는 별 문양

### 선택 인터랙션
- 카드 탭 → 테두리 골드 글로우 (선택 표시)
- 선택된 카드를 다시 탭 → 선택 해제
- 3장 선택 완료 시 → 하단에 "카드 열기" CTA 버튼 등장
- 3장 미만 선택 시 CTA 비활성

---

## Stage 2: 카드 공개

### 플립 애니메이션
- "카드 열기" 탭 → 선택된 3장이 0.3초 간격으로 순차 Y축 180° 회전 (0.6초/장)
- 미선택 2장: 동시에 어두워짐 (opacity 0.3) + 🔒 자물쇠 아이콘 오버레이

### 카드 앞면 정보
국기 + 도시명(영/한) + 핵심 지표 3개:
- 💰 월 생활비 (USD)
- 🛂 무비자 N일 (또는 "비자 필요")
- 📶 인터넷 속도 (Mbps)

### 잠금 카드 (2장)
- 시각적: 어두운 오버레이 + 🔒
- 데이터: 백엔드 게이팅 — API 응답에 잠금 카드의 도시명/상세 정보 미포함
- 잠금 카드 탭 시: "이 카드는 아직 공개할 수 없어요" 또는 유료 결제 안내 (추후)

### 리딩 선택
- 부채꼴 레이아웃 그대로 유지
- 안내 텍스트: "끌리는 도시를 하나 골라보세요"
- 공개된 3장 중 1장 탭 → 선택 글로우 → "리딩 받기" CTA 등장
- CTA 탭 → OAuth 로그인 체크
  - 로그인 상태: 리딩 페이지로 전환
  - 미로그인: OAuth 로그인 → 리딩 페이지

---

## Stage 3: 타로 리딩

### 진입
- POST /api/detail 호출 (기존 파이프라인)
- 로딩: "카드를 읽고 있어요..." (LLM 응답 대기)

### 톤
- **타로 리더 1인칭 서술**: "리스본 카드가 당신을 선택했네요. 대서양을 마주한 이 도시는..."
- 도입부 1~2문장은 감성적 서술, 이후 실용 정보를 섹션별로 제공

### 콘텐츠 구성
LLM이 생성하는 상세 가이드 (기존 /api/detail 응답):
1. 도입 (타로 리더 톤, 1~2문장)
2. 비자 가이드
3. 월 예산 breakdown
4. 세금 주의사항
5. 현지 생활 팁
6. 숙소 딥링크

### LLM 프롬프트 변경
기존 `system_en.py` / `system.py`의 Step 2 프롬프트에 타로 리더 톤 지시 추가.
"마치 타로 리더가 카드를 해석하듯, 따뜻하고 개인적인 톤으로 도시를 소개하라."

### 리딩 제한
- 리딩은 최초 선택한 1장에 대해서만 제공
- 나머지 2장은 리딩 불가 (비교 뷰에서 카드 정보만 확인)

---

## Stage 4: 3장 비교 뷰

### 진입
- 리딩 페이지 하단 또는 리딩 완료 후 자연스럽게 전환
- "다른 카드도 살펴볼까요?" CTA

### 레이아웃
- 공개된 3장을 나란히 배치
- 데스크톱: 3장 가로 배치
- 모바일: 세로 스크롤
- 현재 result/page.tsx의 카드 정보 수준 유지

### 카드 정보
각 카드에 표시되는 정보 (기존 TOP3 결과 카드와 동일):
- 도시명 + 국기
- 비자 배지 (무비자 N일 / 셴겐 / 비자 필요)
- 비자 유형 + 체류 기간 + 갱신 여부
- 월 생활비
- 인터넷 속도
- 안전 점수
- 영어 통용도
- 한인 커뮤니티
- 숙소 딥링크
- 도시 소개 텍스트

## 백엔드 변경

### /api/recommend
- `top_n` 기본값: 3 → 5
- 응답 구조 변경:

```json
{
  "revealed_cities": [
    { "city": "Lisbon", ...전체 정보... },
    { "city": "Chiang Mai", ...전체 정보... },
    { "city": "Medellín", ...전체 정보... }
  ],
  "locked_cities": [
    { "slot": 4 },
    { "slot": 5 }
  ],
  "session_id": "abc123"
}
```

- 잠금 카드(locked_cities): 도시명, 점수, 상세 정보 일체 미포함
- `session_id`: 어떤 5장이 추천되었는지 서버 세션에 저장 (결제 시 잠금 해제용)

현재는 프론트엔드에서 5장 중 유저가 선택한 3장의 인덱스를 보내는 방식.
선택하지 않은 2장의 데이터는 초기 응답에 포함하지 않음.

### 추천 → 공개 2단계 API
1. `POST /api/recommend` → 5장 생성, 세션 저장, 잠금 상태로 응답
2. `POST /api/reveal` (신규) → 유저가 선택한 3장의 인덱스를 보내면, 해당 3장의 상세 데이터 반환

이 방식으로 백엔드 게이팅 구현.

### /api/detail
- 기존 파이프라인 유지
- LLM 프롬프트에 타로 리더 톤 추가

## 프론트엔드 변경

### result/page.tsx 전면 재설계
현재 즉시 3장 카드 표시 → 타로카드 4-Stage 플로우로 교체

### 주요 컴포넌트
- `TarotDeck`: 부채꼴 레이아웃 + 셔플 애니메이션
- `TarotCard`: 뒷면/앞면 + 플립 애니메이션 + 선택 글로우 + 잠금 오버레이
- `TarotReading`: 리딩 결과 표시 (LLM 마크다운)
- `CityCompare`: 3장 비교 뷰 (기존 카드 정보 재활용)

### 애니메이션 (Framer Motion)
- 덱 stagger: `transition={{ delay: index * 0.3 }}`
- Y축 플립: `rotateY` 0° → 180° (0.6초)
- 글로우: `boxShadow` 애니메이션
- 잠금: `opacity` 0.3 + 🔒 아이콘 fade in

### 상태 관리
```
stage: "deck" | "selecting" | "revealed" | "reading" | "comparing"
selectedIndices: number[] (최대 3)
revealedCities: CityData[] (API에서 받은 3장)
readingCity: number (리딩 선택한 카드 인덱스)
readingMarkdown: string (LLM 응답)
```

## BM (향후)

- 잠긴 2장 추가 공개: 결제 → `POST /api/unlock` → session_id 기반으로 잠긴 2장 반환
- 추가 리딩: 결제 → 공개된 카드에 대해 /api/detail 추가 호출 가능
- 현재 구현에서는 잠금 카드 탭 시 "아직 공개할 수 없어요" 안내만 표시
